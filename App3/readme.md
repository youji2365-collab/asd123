# App3 - WinUI 3 애플리케이션

이 애플리케이션은 사용자 정의 아이콘과 대화형 호버 효과, 그리고 이미지 선택 기능을 시연하는 WinUI 3 샘플 애플리케이션입니다.

## 기능

*   **사용자 정의 아이콘:** 창 중앙에 원형 아이콘이 표시됩니다.
*   **호버 효과:**
    *   아이콘 위에 마우스를 올리면 "RE" 텍스트가 페이드인됩니다.
    *   아이콘의 밝기가 50% 감소됩니다 (50% 불투명한 검은색 오버레이가 페이드인).
    *   마우스가 아이콘 영역을 벗어나면 두 효과 모두 페이드아웃됩니다.
*   **아이콘 변경 기능:** 메뉴바의 "Actions" > "Change Icon" 메뉴 항목을 통해 사용자가 새 이미지 파일(`.jpg`, `.jpeg`, `.png`)을 선택하여 아이콘을 업데이트할 수 있습니다.
*   **기본 아이콘 대체:** 기본 `Assets/images.png` 파일이 없을 경우, 회색 원형 플레이스홀더가 표시됩니다.

## 필수 구성 요소

이 애플리케이션을 빌드하고 실행하려면 다음이 필요합니다.

*   **Visual Studio 2022 (또는 최신 버전):**
    *   "C++를 사용한 데스크톱 개발" 워크로드가 설치되어 있는지 확인하십시오.
    *   "유니버설 Windows 플랫폼 개발" 워크로드가 설치되어 있는지 확인하십시오.
    *   "Windows App SDK C++ 템플릿"이 설치되어 있는지 확인하십시오 (일반적으로 UWP 워크로드의 일부이거나 개별 구성 요소로 사용 가능).
*   **Windows App SDK:** 프로젝트는 Microsoft.WindowsAppSDK를 사용합니다.
*   **Windows 10/11:** 개발은 일반적으로 최신 버전의 Windows에서 이루어집니다.

## 프로젝트 빌드

1.  **Visual Studio에서 열기:**
    *   Visual Studio에서 `App3.sln` 솔루션 파일을 엽니다.
2.  **구성 선택:**
    *   Visual Studio 도구 모음에서 원하는 빌드 구성(예: `Debug` 또는 `Release`) 및 플랫폼(예: `x64`)을 선택합니다.
3.  **솔루션 빌드:**
    *   `빌드` > `솔루션 빌드` (또는 `Ctrl+Shift+B` 누르기)를 선택합니다.
    *   빌드 오류가 없는지 확인하십시오. 오류가 발생하면 `빌드` > `솔루션 정리` 후 `빌드` > `솔루션 다시 빌드`를 시도하십시오.

## 애플리케이션 실행

1.  **시작 프로젝트로 설정:**
    *   솔루션 탐색기에서 `App3` 프로젝트를 마우스 오른쪽 버튼으로 클릭하고 `시작 프로젝트로 설정`을 선택합니다.
2.  **실행:**
    *   Visual Studio 도구 모음에서 `로컬 Windows 디버거` 버튼(녹색 재생 아이콘)을 클릭하거나 `F5`를 누릅니다.
    *   애플리케이션 창이 나타나야 합니다.

## 사용법

*   **아이콘 변경:** 메뉴바에서 `Actions` > `Change Icon`을 클릭하여 파일 선택기를 열고 새 이미지를 선택합니다.
*   **호버 효과:** 원형 아이콘 위로 마우스 포인터를 이동하여 "RE" 텍스트 및 밝기 감소 효과를 확인하십시오.

## 문제 해결

*   **빌드 오류:** 변경 사항 적용 후에도 지속적인 빌드 오류가 발생하면 Visual Studio에서 `솔루션 정리` 후 `솔루션 다시 빌드`를 수행해 보십시오.
*   **누락된 에셋:** `App3/Assets` 폴더에 `Assets/images.png` 파일이 있는지 확인하십시오. 파일이 없으면 아이콘은 기본적으로 회색 원으로 표시됩니다.
*   **런타임 오류:** 애플리케이션이 런타임에 충돌하면 모든 필수 구성 요소가 올바르게 설치되었고 프로젝트가 올바른 플랫폼(`x64`가 일반적)용으로 빌드되었는지 확인하십시오.
